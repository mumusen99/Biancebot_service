# 自动交易策略约束文档 v3

> 基于 `auto_trader.py` + `trader.py` + `indicators.py` 的完整交易规则。
> 运行模式：**实盘** (IS_TESTNET=false) | 每15分钟一个心跳周期。
> **v3 改进**: 加入BTC趋势过滤器、ATR动态止损、超卖陷阱防护、风控熔断。

## 1. 资金管理规则

### 1.1 预算上限

| 参数 | 值 |
|------|-----|
| 初始预算 (INITIAL_BUDGET) | **20 USDT** |
| 单次开仓上限 (MAX_POSITION_USDT) | 20 USDT (主流) / 10 USDT (山寨) |
| 总仓位上限 (MAX_TOTAL_POSITION_USDT) | 50 USDT |

> **主流币**: BTC/USDT, ETH/USDT, SOL/USDT, BNB/USDT, XRP/USDT (及前十市值)
> **山寨币**: 除了主流币以外的所有币种
| 杠杆倍数 (DEFAULT_LEVERAGE) | **5x** |
| 保证金模式 | **全仓** (ISOLATED_MARGIN=false) |
| 最小名义价值 | 5 USDT (Binance 合约规则) |

### 1.2 仓位管理

- **最大同时持仓数**: **6个**（v3新增）超过时不再开新单，待老单平仓释放位置
- **每单金额**: 由决策模块动态计算
  - `per_trade_max = max(5, remaining / slots_left)` 其中 `slots_left = max(1, 5 - 当前持仓数)`
  - 单笔最小 10U（除非余额不够时 5U 也接受）
- **换仓条件**: 新机会评分高出最差持仓 5 分以上，或最差持仓亏损 > -3%
- **日亏损熔断 (v3)**: 单日总 PnL < -5U 时，**当日停止开新单**，只允许管理已有持仓。次日重置。
- **连续亏损保护 (v3)**: 最近3笔交易全部亏损 → 仓位系数降为 0.5x，评分门槛+3

### 1.3 停机规则

- 总 PnL ≤ -20 USDT 时**永久停机**，发紧急通知
- 停机后任何周期都不再开仓
- 停机原因写入 `bot_state.json` 的 `stop_reason`

## 2. 风控参数

### 2.1 ATR 动态止损 + 三档分批止盈

**ATR 动态止损（替代固定百分比止损）：**

固定止损在熊市中容易被随机波打掉。改用 ATR(14) 动态计算：

- LONG 止损 = entry - ATR × **1.5**（4h周期）
- SHORT 止损 = entry + ATR × **1.5**
- ATR 从 4h K线计算（最近50根）
- 若 ATR 数据不可用，回退到 5% 保证金止损
- 最大止损宽度不超过 8% 保证金（约束极端行情）

**分三档止盈（半仓先出，剩余继续跑）：**

| 档位 | 止盈位 (保证金%) | 卖出比例 | 换算价格变动 (5x) | 触发后动作 |
|------|-----------------|---------|------------------|-----------|
| TP1 🥇 | 8%~12%（先锁） | **50%** 仓位 | ≈ **1.6%~2.4%** | 上移止损到入场价 🛡️ |
| TP2 🥈 | 15%~20%（中线）| **25%** 仓位 | ≈ **3%~4%** | — |
| TP3 🥉 | 25%~30%（冲刺）| **25%** 仓位 | ≈ **5%~6%** | — |

> 💡 例：BTC 开多 $60,000，止损 $58,800（5x杠杆2%价格=10%保证金）。
> 分三单：TP1 @$61,200（50%出场，成交后止损上移到$60,000）、TP2 @$62,400（25%出场）、TP3 @$63,600（25%出场）

### 2.2 追踪止盈 (动态移动止损)

当持仓产生浮盈时，止损位逐步上移/下移以锁定利润：

| 浮盈区间 (PnL%) | LONG 止损位 | SHORT 止损位 |
|---|---|---|
| > 8% | 入场价 + 2% 价格 | 入场价 - 2% 价格 |
| > 5% | 入场价 + 1% 价格 | 入场价 - 1% 价格 |
| > 3% | 入场价 + 0.33% 价格 | 入场价 - 0.33% 价格 |
| 0% ~ 3% | 入场价 - 0.1% 价格 | 入场价 + 0.1% 价格 |
| 亏损中 | 入场价 - 1.67% 价格（原始止损） | 入场价 + 1.67% 价格 |

> ⚠️ 只有当新止损与旧止损差值 > 0.3% 入场价时才实际移动，避免频繁修改。

### 2.3 订单安全规则

**核心原则：先挂止损和三档止盈，再市价开仓。**

```
开仓流程:
1. 获取当前价格
2. 计算总数量，均分3份，对齐到交易所 stepSize
3. POST algoOrder (STOP_MARKET) → 挂止损 (全仓)
4. POST algoOrder (TAKE_PROFIT_MARKET) → 挂TP1 (1/3仓位)
5. POST algoOrder (TAKE_PROFIT_MARKET) → 挂TP2 (1/3仓位)
6. POST algoOrder (TAKE_PROFIT_MARKET) → 挂TP3 (1/3仓位)
7. ✅ 以上全部成功后 → 市价单开仓 (MARKET BUY/SELL)
8. ❌ 任何一步失败 → 不开仓，返回错误
```

> 止盈止损使用 **FAPI Algo Order** (`/fapi/v1/algoOrder`)，基于标记价格 (MARK_PRICE) 触发。
> 三个止盈单分别挂不同价格，各占 1/3 仓位。
> TP1先出场锁定利润，TP2和TP3让利润继续奔跑。

## 3. 技术指标策略

### 3.1 使用的指标

| 指标 | 参数 | 权重 (评分制) |
|---|---|---|
| EMA (指数移动平均) | 快线 9, 慢线 21 | +2 (金叉/死叉) +1 (排列方向) |
| RSI (相对强弱指标) | 14周期, 超卖30/超买70 | +2 (超卖/超买) +2 (背离) | ⚠️ RSI<30连续>6h → 扣-2分 |
| MACD (异同移动平均) | 快12/慢26/信号9 | +2 (金叉/死叉) +1 (柱翻正/翻负) |
| 布林带 | 20周期, 2标准差 | +1.5 (触轨) +1 (收窄变盘) |
| 成交量 | 20日均量对比 | +1 (放量上涨/下跌) |
| 5日趋势 | 最近5根K线涨跌幅 | +1 (涨>2%/跌>2%) |

### 3.2 评分体系

- `long_score` / `short_score` 各在 **0 ~ 10** 范围内
- 综合评分 = 5m信号评分 + 15m信号评分（范围 0 ~ 20）
- 趋势方向判断:
  - `long_score > short_score + 2` → **LONG**
  - `short_score > long_score + 2` → **SHORT**
  - 否则 → **NEUTRAL**

### 3.3 超卖陷阱防护 (v3)

RSI < 30 连续超过 **6根1hK线**后，取消RSI超卖的+2加分，**改为扣-2分**。

```python
if rsi < 30:
    if oversold_count_1h >= 6:
        long_score -= 2  # 扣分！
    else:
        long_score += 2  # 正常超卖加分
    oversold_count_1h += 1
elif rsi > 30:
    oversold_count_1h = 0  # 重置
```

> 原理：正常回调超卖 → RSI<30后很快反弹；陷阱性超卖 → RSI<30十几个小时不反弹，说明趋势彻底转空。SOXL就是典型，RSI在15-25区间耗了20小时还在跌。

### 3.4 BTC 趋势过滤器（v3新增）

**核心逻辑：** 大盘方向决定是否能逆势开单，避免在下跌趋势中抄底。

```
1. 获取 BTCUSDT 4h 级别 EMA9/EMA21 排列
2. BTC方向判断:
   - BTC 4h EMA9 > EMA21 +  布林中轨以上 → BULLISH
   - BTC 4h EMA9 < EMA21 +  布林中轨以下 → BEARISH
   - 其他 → NEUTRAL

3. 方向约束:
   BTC BULLISH  → 多单正常，空单门槛+3分
   BTC BEARISH  → 空单正常，多单门槛+3分，仓位×0.5
   BTC NEUTRAL  → 正常

4. 跳过规则:
   BTC 单日跌幅 > 5%  → 全天不开任何多单
   BTC 单日涨幅 > 5%  → 全天不开任何空单
```

> v3 之前的问题：不看大盘，牛市做多熊市也做多，熊市全部亏回去。
> v3 改进：大盘空头时做多门槛从12分提升到15分，仓位减半，减少抄底亏损。

## 4. 选币逻辑

### 4.1 全市场扫描

```
全市场 USDT 合约 (648个)
     ↓ 按24h成交量排序取前60名
     ↓ 按24h涨跌幅绝对值排序取前20
     ↓ 合并去重 + 必须包含配置中16个币
     ↓ 对每个候选做技术分析
     ↓ 按综合评分排序 (score_long/score_short + 关注度)
```

### 4.2 重点关注币种 (SYMBOLS)

BTCUSDT, ETHUSDT, BNBUSDT, SOLUSDT, XRPUSDT, DOGEUSDT, ADAUSDT, AVAXUSDT, LINKUSDT, DOTUSDT, MATICUSDT, NEARUSDT, APTUSDT, ARBUSDT, OPUSDT, PEPEUSDT

### 4.3 风险评分

| 24h涨跌幅 | 风险分 |
|-----------|--------|
| > 15% | +3 (高风险) |
| 8% ~ 15% | +2 |
| 4% ~ 8% | +1 |
| < 4% | 0 |

风险 ≥ 3 的币**不进入开仓候选项**。

## 5. KOL 情绪评分

### 5.1 数据来源

AI 通过 `web_search` 手动搜集 KOL 动态写入 `sentiment.json`。

### 5.2 情绪影响

| 情绪 | 评分修正 | 行为 |
|------|----------|------|
| `bullish` | LONG +1.5, SHORT -1.5 | 倾向做多 |
| `bearish` | LONG -1.5, SHORT +1.5 | 倾向做空 |
| `neutral` | 无修正 | 正常 |

### 5.3 风险覆盖

| 条件 | 仓位系数 | 最低评分门槛 | 行为 |
|------|----------|--------------|------|
| 空头 > 多头 +30% | 0 (不开仓) | 99 (不可能达成) | 完全跳过 |
| 空头 > 多头 +10% | 0.6 (降仓) | 14 | 降低仓位 |
| 正常 | 1.0 | 12 | 正常交易 |

## 6. 决策流程

### 6.1 完整周期 (run_cycle)

```
1. 加载 bot_state.json → 读取状态
2. sync_positions → 同步交易所上的实时持仓
3. manage_positions → 检查止盈/止损/追踪止盈
4. check_stop → 检查是否触发停机
5. scan_with_risk → 全市场扫描+技术分析
6. decide → 综合决策 (多空评分 + KOL + 风险)
7. 执行:
   - HOLD → 什么都不做
   - OPEN → safe_open_position() 先SL/TP再开仓
   - REPLACE → 先平旧仓再开新仓
8. 保存 bot_state.json
```

### 6.2 开仓条件 (综合门槛)

多空**平等竞争**，每个候选币取多空评分较高的方向作为推荐方向。

```
候选池 = 全市场648 → 成交量+涨跌幅筛选 → 技术深析 → 按 max(long,short) + 关注度 排序

对每个候选币:
  1. 方向 = long >= short ? LONG : SHORT
  2. 有效分 = max(score_long, score_short)  (KOL情绪不再修正分数)
  3. 有效分 >= 12  (风险预警时提升到 14+)
  4. 推荐方向必须比反方向高 3 分以上 (排除模糊信号)
  5. 高波动币 (24h涨跌幅TOP5) 需要 >= 15 分才开
  6. 交易所上已有该币持仓则跳过

满足条件就开，第一个达标的候选币直接执行，不再区分多空优先级。
```

> 旧逻辑：先遍历所有做多候选 → 有合格的直接返回 → 才看做空。
> 新逻辑：统一排序，谁分数高开谁，多空完全平等。

### 6.3 换仓条件

满足以下任一即触发换仓：

1. 最差持仓亏损 `pnl_percent < -3%`
2. 新机会评分 - 旧持仓评分 ≥ **5分**

换仓逻辑：先平仓 → 取消该币 algo订单 → (等待1秒) → 开新仓

## 7. 持仓同步规则

### 7.1 同步机制

每次周期从交易所 `positionRisk` API 拉取实时持仓，和 `bot_state.json` 比对：

- 交易所上有的但 bot_state 里没有的 → 视为外部/手动开仓，**跳过不碰**
- bot_state 里有但交易所已平的 → 从bot记录中删除
- 更新实时价格、浮盈浮亏到 state

### 7.2 用户手动单保护

`USER_MANUAL_SYMBOLS` 集合中定义的币种，bot 永不开仓/平仓/修改。

（目前为空集，后续可追加如 DOGEUSDT）

## 8. 自检与修复 (auto_check.py)

### 8.1 止盈止损自检

每周期检查每个持仓是否有有效的 SL/TP（Algo Order 状态为 NEW）：

- **缺失止损或止盈** → 重新挂单
- **价格偏差 > 0.5%** → 取消旧单，重新挂
- **重复订单** → 清理多余的

### 8.2 通知队列

Python 脚本通过 `notifications.py` 写入 `notifications.json`，AI 心跳周期读取并推送到微信。

通知类型:

| Type | 场景 | 推送级别 |
|------|------|----------|
| `open` | 新开仓成功 | 通知 |
| `tp` | 止盈平仓 | 通知 |
| `sl` | 止损平仓 | 通知 |
| `error` | 开单失败 | 通知 |
| `emergency` | 预算耗尽停机 | 紧急通知 |

## 9. 报错恢复

### 9.1 常见错误处理

| 错误 | 影响 | 处理 |
|------|------|------|
| SL/TP 精度错误 (400) | 该单不开 | 等待下次周期自动对齐精度 |
| 平仓 404 | 换仓失败 | 仓位保留，下次周期重试 |
| 超时 (120s+) | 整周期跳过 | 已改为 180s 超时 |
| 网络错误 | 状态不变 | 下次心跳重试 |

### 9.2 状态持久化

| 文件 | 作用 |
|------|------|
| `bot_state.json` | AI 机器人核心状态 (预算/持仓/盈亏) |
| `state.json` | Trader 类状态 + AI 决策记录 + 模拟持仓 |
| `analysis.json` | Python 分析结果 |
| `notifications.json` | 待发送通知队列 |
| `sentiment.json` | KOL 情绪分析结果 |
