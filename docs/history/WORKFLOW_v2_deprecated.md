## 二、技术引擎 (auto_trader.py) — 每15分钟

### Stage T1: 全市场数据采集

```
请求 /ticker/24hr → 获取全部 USDT 合约 (600+)

┌─ 按24h成交量排序 → TOP 30 (流量币)
├─ 按24h涨跌绝对值排序 → TOP 20 (波动币)  
├─ 合并去重 → 加上用户关注币
└─ 输出: 约40-50个候选币
```

### Stage T2: 技术深析

```
对每个候选币:
  请求K线: 5m×100根 + 15m×100根

  计算: EMA(9/21)金叉死叉
        RSI(14)超买>70/超卖<30 + 背离检测
        MACD(12/26/9)金叉死叉/动量翻转
        布林带(20,2)触轨/收窄变盘
        成交量放量/缩量/倍量

  输出: 多评分 0-10 + 空评分 0-10 + 信号列表
```

### Stage T3: 综合评分 + 风险过滤

```
最终评分 = 技术分(5m+15m) + KOL情绪修正(±1.5)

风险过滤:
  24h波动 > 15% → 禁入
  24h波动 > 8%  → 信号必须≥15
  24h波动 > 4%  → 正常
  KOL风险警告  → 跳过
```

### Stage T4: 决策 — 开仓条件检查

```
┌──────────────────────────────────────────────────┐
│ 前置检查:                                        │
│   IF 机器人状态.stopped → HOLD                   │
│   IF 已有5个AI持仓 → HOLD (达到上限等止盈止损)               │
│   IF 剩余预算 < 5U → HOLD
  IF 预算已用 ≥ 20U → HOLD (满仓)                        │
│   IF 目标在 USER_MANUAL_SYMBOLS 中 → SKIP        │
│   IF sentiment.risk_warning → HOLD               │
│                                                  │
│ ════════════════════════════════════════════════  │
│  模式A: 常规开单 (评分≥12)                        │
│ ════════════════════════════════════════════════  │
│                                                  │
│ 做多审批 (ALL must pass):                        │
│   □ 最佳多候选评分(含KOL修正) ≥ 12               │
│   □ 多分 > 空分 + 3                              │
│   □ 波动大的币: KOL情绪必须一致 ≥bullish          │
│   □ 风险分 < 3                                   │
│                                                  │
│ 做空审批 (ALL must pass):                        │
│   □ 最佳空候选评分(含KOL修正) ≥ 12               │
│   □ 空分 > 多分 + 3                              │
│   □ 风险分 < 3                                   │
│                                                  │
│ 通过 → 进入常规开单流程 (SL-5% / TP+10%)          │
│ 未通过 → 检查是否满足极端行情条件 → HOLD          │
└──────────────────────┬───────────────────────────┘
                       │
                       ▼
```

### Stage T4.5: 极端行情快速交易 (Momentum模式)

```
┌──────────────────────────────────────────────────┐
│ 极端行情检测条件:                                 │
│   1m K线涨跌 ≥ ±3%                               │
│   AND 1m成交量 ≥ 3倍 20周期均值                    │
│   AND 过去15分钟总涨跌 ≥ ±5%                      │
│                                                  │
│ 触发后行为:                                      │
│   ├─ 记录 "⚠️ 极端行情检测" 到日志                 │
│   └─ 进入Momentum模式                            │
│                                                  │
│ Momentum模式规则:                                 │
│   ┌──────────────────────────────────────────┐   │
│   │ 开仓方向:                                  │   │
│   │   暴跌(≥-3%) → 开多 (抄反弹)               │   │
│   │   暴涨(≥+3%) → 开空 (抓回調)               │   │
│   │                                            │   │
│   │ 仓位: 固定 10 USDT (剩余预算够的话)         │   │
│   │                                            │   │
│   │ ⚠️ 止损: -2% (比常规严很多)                │   │
│   │ ⚠️ 止盈: +3% ~ +4% (吃一口就跑)           │   │
│   │                                            │   │
│   │ 禁止: 已有AI持仓时不触发                     │   │
│   │ 禁止: 24h波动>15%的币 (风险过高)            │   │
│   │ 禁止: KOL风险警告激活时                      │   │
│   └──────────────────────────────────────────┘   │
│                                                  │
│ 执行流程与常规开单相同: 先挂SL/TP → 再入场         │
└──────────────────────────────────────────────────┘
```

### Stage T5: 安全开单执行 (两种模式)

```
两种模式共用同一安全流程，仅SL/TP参数不同:

  模式     │ 止损(保证金) │ 止盈(保证金) │ 仓位   │ 适用场景
  ─────────┼──────────────┼──────────────┼────────┼──────────
  常规     │ -5%          │ +10%         │ 剩余   │ 标准技术信号
  Momentum │ -2%          │ +3~4%        │ 10U    │ 极端行情

  ⚠️ SL/TP百分比基于保证金，除以杠杆=真实价格波动
     3x杠杆: 5%止损=价格1.67%, 10%止盈=价格3.33%

┌──────────────────────────────────────────────────┐
│ 第一步: 先挂止损止盈 (Algo API - sapi/v1)         │
│                                                  │
│  止损单:                                         │
│    类型: STOP_MARKET                             │
│    价格: 入场价 × (1 - sl%)   [多头]              │
│          入场价 × (1 + sl%)   [空头]              │
│    positionSide: LONG / SHORT                    │
│    urgency: NORMAL, workingType: MARK_PRICE       │
│                                                  │
│  止盈单:                                         │
│    类型: TAKE_PROFIT_MARKET                      │
│    价格: 入场价 × (1 + tp%)  [多头]              │
│          入场价 × (1 - tp%)  [空头]              │
│    positionSide: LONG / SHORT                    │
│    urgency: NORMAL, workingType: MARK_PRICE       │
│                                                  │
│  ┌────────────────────────────────────────────┐  │
│  │ ✅ SL/TP全部挂单成功 → 进入第二步           │  │
│  │ ❌ 任意挂单失败 → 取消所有，不开仓           │  │
│  └────────────────────────────────────────────┘  │
│                                                  │
│ 第二步: 市价开仓 (FAPI /order)                   │
│    symbol, side(BUY/SELL), type=MARKET           │
│    quantity=精度对齐, positionSide=LONG/SHORT     │
│    最小名义价值 ≥ 5 USDT                         │
│    开仓成功 → 记录到 bot_state.json (标记模式)    │
│    开仓失败 → 取消止损止盈，等下次周期             │
└──────────────────────────────────────────────────┘
```

### Stage T6: 仓位管理 (每周期) — 含动态止盈止损调整

```
每周期对每个 AI 持仓执行:

  ▸ 第一步: 止盈/止损检查
    ┌────────────────────────────────────────┐
    │ PnL ≥ +10%  →  市价平仓 + 记录 + 通知  │
    │ PnL ≤ -5%   →  市价平仓 + 记录 + 通知  │
    │              → 总亏损≥20U → 停机       │
    └────────────────────────────────────────┘

  ▸ 第二步: 动态止损调整 (追踪止盈)
    检查当前 PnL，决定是否上移止损:

    ┌────────────────────────────────────────┐
    │ 当前 PnL     │ 止损移至 (常规/Momentum)  │
    ├────────────────────────────────────────┤
    │ < 0% (亏损)  │ 不动                     │
    │ 0% ~ 3%      │ 保本微利 (入场-0.3%)    │
    │ 3% ~ 5%      │ 锁1%利润                │
    │ 5% ~ 8%      │ 锁3%利润                │
    │ > 8%         │ 锁6%利润                │
    │ Momentum单   │ 做多: PnL≥2%即移动止损  │
    │              │ 至保本 (+0.5%)          │
    │              │ 吃到3-4%即刻止盈        │
    └────────────────────────────────────────┘

    每次调整前: 比较新止损与上次调整值
    差价 < 0.3% → 跳过 (避免频繁修改)

    调整流程:
    1. 取消当前 Algo 止损单
    2. 在 Algo API 下新止损单
    3. 更新 bot_state.positions[].last_sl
```

### Stage T7: 停止条件

```
停机条件 (任一满足):
  ⚠️ 总PnL ≤ -20 USDT (预算亏完)
  ⚠️ bot_state.stopped = true

停机后:
  ├─ cron 继续运行但不执行(跳过分析)
  ├─ 不清除已有持仓 (等Algo自动平)
  └─ 等待用户手动指令重启
```

## 三、KOL情绪引擎 (AI执行) — 每4小时

### Stage K1: 触发检测

```
AI上线时:
  检查 /workspace/.kol_check_due 是否存在
  存在 → 进入KOL分析流程
  不存在 → 跳过 (未到4小时周期)
```

### Stage K2: 多来源搜索 (web_search) — 时效性严格检查

```
⚠️ 核心原则: 只采信 4 小时内的消息，过期一律丢弃

搜索策略 (固定4个维度):

  1. 整体币圈情绪
     → 搜: "crypto market sentiment today"  (freshness=day)
     → 搜: "比特币 市场情绪" (freshness=day)

  2. KOL/博主观点
     → 搜: "crypto influencers market outlook"
     → 搜: "加密货币 大V 行情分析"

  3. 宏观经济影响
     → 搜: "美联储 crypto market" (freshness=day)
     → 搜: "crypto regulation" (freshness=week)

  4. 特定币种热度
     → 搜: 当前涨跌榜TOP3币的动态 (freshness=day)
```

### Stage K3: 时效性过滤 + 情绪分析

```
对 web_search 返回的每条结果:

  ┌────────────────────────────────────────┐
  │ 检查发布时间:                           │
  │                                        │
  │  发布时间 < 4小时前 → ✅ 采纳           │
  │  发布时间 ≥ 4小时前 → ❌ 丢弃           │
  │  无发布时间标记 →  默认丢弃             │
  │                                        │
  │  ⚠️ 如果采纳的消息不足 3 条:            │
  │      → 跳过KOL情绪分析                 │
  │      → 保留上次情绪数据                 │
  │      → 日志记录

### Stage K4: 情绪量化评分 (只统计已过时效性检查的消息)

```
对过审的每条消息执行情绪分析:

  看多方信号:
    - "看多/买入/抄底/bullish/buy the dip"
    - "利好/升级/合作/减半/halving/ETF"
    - "目标价/突破/resistance broken"

  看空方信号:
    - "看空/卖出/逃顶/bearish/sell"
    - "利空/监管/黑客/restriction"
    - "回调/出货/dump/liquidation"

  结果计算:
    看多数量 / 有效消息总数 × 100 = bullish%
    看空数量 / 有效消息总数 × 100 = bearish%
    中性 = 100 - bullish% - bearish%

    总体倾向: bullish% > bearish% + 20% → "bullish"
              bearish% > bullish% + 20% → "bearish"
              否则 → "neutral"

    风险警告: 出现以下关键词触发
      - "危机/crisis/crash/暴跌/崩盘"
      - "战争/制裁/sanctions/禁令/ban"
      - "加息/紧缩/hawkish"

    过期率: 被丢弃的过期消息 / 总搜索结果
    如果过期率 > 60% → 本次分析结果不可靠，维持上次
```

### Stage K5: 写入 sentiment.json

```
{
  "last_update": "2026-07-03T23:20:00",
  "overall_sentiment": "bullish" | "bearish" | "neutral",
  "details": {
    "bullish_count": 8,
    "bearish_count": 3,
    "neutral_count": 4
  },
  "key_topics": ["ETF", "减半", "L2"],
  "risk_warning": false,
  "sources": ["twitter", "微博", "新闻"],
  "analysis_text": "当前市场整体偏多..."
}
```

### Stage K6: 情绪修正注入技术引擎

```
auto_trader 读取 sentiment.json:

  sentiment == "bullish"  → 做多评分 +1.5
  sentiment == "bearish"  → 做空评分 +1.5
  sentiment == "neutral"  → 无修正

  risk_warning == true    → 禁止开新单
```

## 六、安全底线 (不可违反)

```
1. ❗ 先挂止盈止损 → 再开仓 (绝不能裸奔)
2. ❗ 预算 20 USDT 刚性上限 → 亏完自动停机
3. ❗ 用户手动单 (DOGEUSDT) → 永久跳过绝不碰
4. ❗ 高风险币 (24h波动>15%) → 不追
5. ❗ KOL风险警告激活时 → 不开新单
6. ❗ 每4小时必须刷新一次KOL情绪
```
