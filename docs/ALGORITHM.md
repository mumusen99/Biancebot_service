## 一、信号评分系统

每个候选币扫描 5m K 线（60 根），计算多维指标后打分，满分约 16 分。

### 1.1 评分维度

| 维度 | 最高分 | 说明 |
|------|--------|------|
| **趋势结构** | 4.0 | EMA9/21 排列、价格在 EMA 上方/下方、前一根 K 线同向确认 |
| **EMA 回踩质量** | 2.0 | 价格距 EMA9 的距离：理想≤0.25%→2.0，正常≤0.50%→1.5，允许≤0.70%→1.0 |
| **RSI** | 2.0 | 理想区间(LONG:44-54, SHORT:46-56)→2.0，允许区间→1.0，边缘区→0.5 |
| **MACD** | 2.5 | 金叉+1.5，零轴上方+0.5，柱状图放大+0.5（仅加分不扣分） |
| **K 线确认** | 2.5 | 当前价>前收+1.0，当前价>前EMA9+1.0，影线方向确认+0.5 |
| **成交量** | 2.0 | 大于 5 日均量 80%→2.0，50%→1.5，35%→1.0 |
| **BTC 环境** | 1.5 | bias≥4→1.5, bias≥1→0.5（方向一致时加分） |
| **订单流** | 1.0 | 买卖盘深度比≥1.3→+1.0 |
| **回撤质量** | ±1.5 | 健康缩量+1.0，放量回调-1.0，回调K线数2-8根正常 |
| **Higher Low** | ±1.0 | LONG出现Higher Low→+1.0, Lower Low→-1.0 |

### 1.2 硬拒绝（直接跳过）

以下情况**直接放弃**，不进入评分：

- EMA 距离 > 1.0%（硬拒绝线）
- 成交量比 < 35%（硬拒绝线）
- EMA9 孤立（距 EMA9≤0.15% 但 VWAP/EMA20 都远 → 假支撑，-1.5 分）
- 距前高/前低 < 0.3×ATR（没有空间）
- VWAP 偏离 > 1.2×ATR（追高/追低）
- 动量衰竭：RSI 极端 + 连续缩量

### 1.3 极值位置惩罚

| LONG pos% | SHORT pos% | 惩罚分 |
|-----------|------------|--------|
| >90% | <10% | -20 |
| >80% | <20% | -12 |
| >70% | <30% | -5 |
| 35%-65% | 35%-65% | -8（区间中部） |

## 二、入场分层（Tier Routing）

根据最终评分将信号路由到不同执行方式：

```
⭐ ≥6.5  → MARKET 市价（position在有利位置时）
            ↓ 如果position极端(>65%LONG/<35%SHORT) → 降级为 aggressive LIMIT
⭐ 5.5-6.5 → aggressive LIMIT（90s TTL，贴近市价）
⭐ 4.5-5.5 → passive LIMIT（300s TTL，较远挂单）
⭐ <4.5   → skip（跳过）
```

### 2.1 四维门槛（按交易类型）

每个信号计算 dir/loc/trig/exec 四个子维度，按交易类型校验：

| 交易类型 | dir | loc | trig | exec | 触发条件 |
|----------|-----|-----|------|------|----------|
| TREND_PULLBACK | ≥4.5 | ≥3.5 | ≥4.0 | ≥4.0 | 趋势+0.20≤pos≤0.60 |
| RANGE_REVERSAL | ≥3.0 | ≥5.0 | ≥4.5 | ≥4.0 | 震荡+极值位 |
| MOMENTUM_SCALP | ≥4.5 | ≥3.0 | ≥5.5 | ≥5.0 | 趋势高位 |

任一维不达标 → 拒绝。

### 2.2 交易类型判定

```
BTC 趋势模式 (mild_bull/bear/strong_*):
  pos 20%-60%        → TREND_PULLBACK
  pos <20%           → RANGE_REVERSAL（低位反转）
  pos >60%           → MOMENTUM_SCALP（高位仅快单）

BTC 震荡模式 (range/CHOP):
  pos ≤25% 或 ≥75%   → RANGE_REVERSAL
  25%<pos<75%        → 禁止开仓（CHOP 区间中部）

过渡态:
  仅 pos ≤20% 或 ≥80% → RANGE_REVERSAL
```

## 三、入场价格与止损计算

### 3.1 结构止损

取最保守锚点：`structural_sl = min(pullback_low, swing_low, ema21 - 0.2×ATR)`（LONG）
然后再减去 ATR 缓冲（min 0.5×ATR_5m, 0.1%×price）

### 3.2 挂单价

| 方向 | 公式 | 含义 |
|------|------|------|
| LONG | `max(ema9, close × 0.998)` | 买在 EMA 或略低于市价 |
| SHORT | `min(ema9, close × 1.001)` | 卖在 EMA 或略高于市价 |

**激进 LIMIT 调整**（把挂单价拉近市价）：

| 方向 | 公式 |
|------|------|
| LONG | `min(ticker_last × 1.001, signal_limit)` |
| SHORT | `min(ticker_last × 1.0005, signal_limit)` |

### 3.3 止损范围约束

| 交易类型 | 最小止损 | 最大止损 |
|----------|----------|----------|
| TREND_PULLBACK | 0.35% | 0.90% |
| RANGE_REVERSAL | 0.18% | 0.45% |
| MOMENTUM_SCALP | 0.12% | 0.35% |

结构止损太近 → 拉宽到 min；太远（>max）→ 拒绝交易。

### 3.4 仓位计算

```
单笔风险 = min(0.50U, 账户净值 × 0.2%) × 信心系数
名义价值 = min(单笔风险 / (止损% / 100), 20U × 5x)
数量 = 名义价值 / 入场价（对齐精度）
```

连损减仓：连续亏损后 `仓位 × pos_mult`（pos_mult 随连损次数递减）。

### 3.5 追价保护

入场时当前价偏离计划入场价 > 0.15R → 取消。

## 四、引擎止盈止损监控

引擎主循环每 **0.5s** 从 WS 获取实时价格，对所有活跃持仓执行：

### 4.1 止损 SL

```
R = |entry - sl|
SL 条件: LONG: price ≤ sl   SHORT: price ≥ sl
需连续2次确认（防插针）
HIT → 市价全平 → QQ通知
```

### 4.2 分段止盈（方案B）

| 阶段 | 比例 | 触发条件 | 价格 |
|------|------|----------|------|
| TP1 | 50% | ≥2次确认 | `entry + 1.0×R` (LONG) |
| TP2 | 30% | ≥2次确认 | `entry + 1.5×R` (LONG) |
| Runner | 20% | 移动止损 | 见下方 |

TP1 触发后激活 `trailing_active` 标志，剩余仓位进入移动止损模式。

### 4.3 移动止损（Runner）

```
激活点: TP3 = entry + 2.5×R（到达即激活，不到达则不追踪）
激活后: 追踪最高价 highest_price
回撤: highest_price - 0.5×R → 平仓剩余仓位
每次创新高: 重置确认计数器
```

### 4.4 两段确认机制

所有 SL/TP/Trail 都需**连续 2 次**命中才执行，防止插针假信号。确认计数器持久化到状态文件，跨循环不丢失。

## 五、持仓审查退出（_review_scalp_positions）

每次 scan 周期（60s）对所有活跃 scalp 持仓审查，基于生命周期 + 软退出评分：

### 5.1 生命周期阶段

| 阶段 | 持仓时间 | 行为 |
|------|----------|------|
| **protect** | 0-60s | 不退出，仅记录最佳浮盈 |
| **startup** | 60-180s | soft≥3→退出，mfe<0.02%且soft≥1→退出 |
| **observe** | 180-480s | soft≥2→退出，360s mfe<0.15%→退出，240s mfe<0.10%→退出 |
| **harvest** | 480-1500s | 未达0.5R且mfe<0.15%→退出，soft≥2→退出 |
| **hard_cap** | >1500s | 强制退出 |

### 5.2 软退出评分（6项，用已收盘K线判断）

| # | 条件 | 说明 |
|---|------|------|
| 1 | 连续跌破VWAP/EMA20 | LONG连续2根低于支撑 |
| 2 | Lower Low / Higher High | 结构破坏 |
| 3 | EMA9 穿越 EMA20 | 短期趋势反转 |
| 4 | 放量 | 成交量 > 1.3倍均量 |
| 5 | RSI 极端 + 恶化 | LONG RSI<38且下降 / SHORT RSI>62且上升 |
| 6 | 趋势反转 | EMA9方向改变 |

### 5.3 浮盈追踪止损

```
浮盈 ≥ 1.0×risk_pct → SL移到 entry + 0.5×risk%（LONG）
浮盈 ≥ 0.6×risk_pct → SL移到 entry（保本）
```

## 六、风险控制

| 机制 | 参数 |
|------|------|
| 日亏损熔断 | 日亏损 > 账户5% → 停止开仓 |
| 连损减仓 | 连续亏损后仓位递减 |
| 8 仓硬上限 | 总持仓 ≤ 8 |
| 5 仓 scalp 上限 | 超短线 ≤ 5 |
| 垃圾轮换 | 满仓时新信号≥8.5分 → 踢掉最低垃圾分（≤-3分） |
| 追价保护 | 入场偏离 > 0.15R → 取消 |
| 市场结构保护 | RISK_OFF 时禁止开仓 / 低风险时谨慎 |
